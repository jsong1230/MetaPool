## 인수조건

### F-01: 마켓 생성
- [ ] 관리자(owner)만 마켓을 생성할 수 있다
- [ ] 질문(영어), 한국어/중국어/일본어 질문, 카테고리, 베팅 마감 시간, 결과 확정 예정 시간을 입력한다
- [ ] 베팅 마감 시간은 현재 시간보다 미래여야 한다
- [ ] 결과 확정 예정 시간은 베팅 마감 시간보다 이후여야 한다
- [ ] 질문 텍스트가 비어있으면 revert한다
- [ ] 생성 시 MarketCreated 이벤트가 emit된다
- [ ] 마켓 ID는 1부터 순차 증가한다

### F-02: 마켓 카테고리
- [ ] 6개 카테고리(Crypto/Sports/Weather/Politics/Entertainment/Other)가 enum으로 정의된다
- [ ] 마켓 생성 시 반드시 하나의 카테고리를 지정한다
- [ ] getMarket으로 조회 시 카테고리 정보가 포함된다

### F-03: 결과 확정
- [ ] 관리자만 결과를 확정할 수 있다
- [ ] Active 또는 Closed 상태의 마켓만 결과 확정 가능하다
- [ ] 베팅 마감 시간 이후에만 결과를 확정할 수 있다
- [ ] Yes/No/Void 중 하나로 확정한다 (Undecided는 불가)
- [ ] 결과 확정 시 패배 풀에서 플랫폼 수수료(2%)를 계산하여 누적한다
- [ ] Void 확정 시 마켓 상태가 Voided로 변경된다
- [ ] MarketResolved 이벤트가 emit된다

### F-04: 마켓 무효화
- [ ] Void로 결과 확정된 마켓에서 모든 베팅자가 원금을 환불받을 수 있다
- [ ] 환불은 사용자가 claimRefund를 호출하여 받는다 (pull pattern)
- [ ] 이미 환불받은 사용자는 재환불 불가하다
- [ ] RefundClaimed 이벤트가 emit된다

### F-05: Yes/No 베팅
- [ ] Active 상태이고 마감 전인 마켓에만 베팅 가능하다
- [ ] 최소 베팅 금액(100 META) 이상이어야 한다
- [ ] 최대 베팅 금액(100,000 META) 이하여야 한다
- [ ] 베팅 시 msg.value로 META를 전송한다
- [ ] Yes 베팅 시 yesPool 증가, No 베팅 시 noPool 증가
- [ ] 첫 베팅 시 해당 방향 참여자 수(Count) 증가
- [ ] BetPlaced 이벤트가 emit된다
- [ ] 글로벌 pause 상태에서는 베팅 불가하다

### F-06: 추가 베팅
- [ ] 이미 베팅한 마켓에서 같은 방향으로 추가 베팅이 가능하다
- [ ] 추가 베팅 시 기존 금액에 합산된다
- [ ] 반대 방향으로의 베팅은 revert된다
- [ ] 추가 베팅도 min/max 범위를 검증한다

### F-07: 보상 클레임
- [ ] Resolved 상태의 마켓에서만 클레임 가능하다
- [ ] 승리 방향에 베팅한 사용자만 클레임 가능하다
- [ ] 보상 = 원금 + (분배 가능 금액 x 내 베팅 / 승리 풀)
- [ ] 분배 가능 금액 = 패배 풀 - 플랫폼 수수료
- [ ] 이중 클레임은 revert된다
- [ ] Checks-Effects-Interactions 패턴 적용 (claimed = true 먼저 설정)
- [ ] WinningsClaimed 이벤트가 emit된다

### F-08: Void 환불
- [ ] Voided 상태의 마켓에서만 환불 가능하다
- [ ] 베팅 기록이 있는 사용자만 환불 가능하다
- [ ] 베팅 원금 전액을 반환한다
- [ ] 이중 환불은 revert된다
- [ ] RefundClaimed 이벤트가 emit된다

### F-09: 이의제기 기간
- [ ] 결과 확정 후 24시간 이의제기 기간이 자동 설정된다
- [ ] 이의제기 기간 중에는 클레임이 보류된다
- [ ] 이의 수가 임계값(전체 베팅자의 10% 이상)을 초과하면 재심 상태로 전환된다
- [ ] 이의제기 기간이 종료되면 정상 클레임이 가능해진다

### F-10: 이의제기
- [ ] 이의제기 기간 내에만 이의를 제출할 수 있다
- [ ] 이의 제출 시 1,000 META를 스테이킹해야 한다
- [ ] 해당 마켓에 베팅한 사용자만 이의를 제출할 수 있다
- [ ] 이의가 인정되면 스테이킹 금액이 반환된다
- [ ] 이의가 기각되면 스테이킹 금액이 몰수된다
- [ ] DisputeSubmitted 이벤트가 emit된다

### F-11: 긴급 마켓 중단
- [ ] 관리자가 Active 상태의 마켓을 Paused 상태로 전환할 수 있다
- [ ] Paused 상태에서는 베팅이 불가하다
- [ ] 관리자가 Paused 마켓을 새 마감시간과 함께 재개할 수 있다
- [ ] MarketPaused / MarketResumed 이벤트가 emit된다

### F-12: 설정 관리
- [ ] 관리자가 minBet, maxBet, platformFeeRate를 변경할 수 있다
- [ ] minBet > 0, maxBet > minBet, feeRate <= 1000(10%) 검증
- [ ] 관리자가 누적 수수료를 인출할 수 있다
- [ ] SettingsUpdated / FeesWithdrawn 이벤트가 emit된다

### F-13: 지갑 연결
- [ ] MetaMask가 설치되지 않은 경우 설치 안내를 표시한다
- [ ] "지갑 연결" 버튼 클릭 시 MetaMask 연결 요청이 뜬다
- [ ] Metadium 네트워크가 아닌 경우 네트워크 전환을 요청한다
- [ ] 연결 후 지갑 주소(축약)와 META 잔액을 헤더에 표시한다
- [ ] 계정/네트워크 변경 이벤트를 감지하여 UI를 업데이트한다

### F-14: 마켓 목록
- [ ] 활성(Active) 마켓을 카드 형태로 나열한다
- [ ] 각 카드에 질문/카테고리 태그/풀 규모/참여자 수/마감 카운트다운/배당률을 표시한다
- [ ] 마켓이 없을 때 빈 상태 메시지를 표시한다
- [ ] 컨트랙트에서 marketCount를 읽어 순차 로딩한다
- [ ] BetPlaced 이벤트를 리스닝하여 풀/배당률을 실시간 갱신한다

### F-15: 카테고리 필터
- [ ] 전체/Crypto/Sports/Weather/Politics/Entertainment/Other 필터 탭을 표시한다
- [ ] 필터 선택 시 해당 카테고리의 마켓만 표시한다
- [ ] 기본값은 "전체"이다

### F-16: 마켓 상세
- [ ] 마켓 카드 클릭 시 상세 화면으로 이동한다
- [ ] 질문, 카테고리, 마감 카운트다운을 표시한다
- [ ] Yes/No 풀 비율을 프로그레스 바로 시각화한다
- [ ] 총 풀 금액, Yes/No 참여자 수를 표시한다
- [ ] Yes/No 각각의 배당률을 표시한다
- [ ] 마감된 마켓은 "베팅 마감" 상태를 표시한다
- [ ] Resolved/Voided 마켓은 결과와 클레임 버튼을 표시한다

### F-17: 실시간 배당률
- [ ] getOdds 함수로 현재 배당률을 조회하여 표시한다 (소수점 2자리)
- [ ] BetPlaced 이벤트 수신 시 배당률을 즉시 갱신한다
- [ ] 풀이 0인 경우 배당률 대신 "첫 번째 베터가 되세요" 표시

### F-18: 베팅 수량 입력
- [ ] Yes 또는 No 베팅 버튼 클릭 시 베팅 패널이 슬라이드 오픈된다
- [ ] 슬라이더로 금액을 조절할 수 있다 (100~100,000 META)
- [ ] 직접 입력 필드로 정확한 금액을 입력할 수 있다
- [ ] 퀵 버튼(100/500/1K/5K/10K META)으로 빠르게 설정할 수 있다
- [ ] 지갑 잔액 초과 시 경고를 표시한다

### F-19: 예상 수익 표시
- [ ] 금액 입력 시 calculatePotentialWinnings로 예상 수익을 실시간 계산한다
- [ ] "예상 수익: X META (X.XXx)" 형태로 표시한다
- [ ] 수수료 차감 후 순수익을 표시한다

### F-20: 베팅 확인 모달
- [ ] 베팅 확인 버튼 클릭 시 확인 모달이 표시된다
- [ ] 모달에 마켓 질문, 선택 방향(Yes/No), 베팅 금액, 현재 배당률, 예상 수익을 표시한다
- [ ] "서명 & 베팅" 클릭 시 MetaMask 트랜잭션 서명을 요청한다
- [ ] 트랜잭션 성공 시 성공 토스트를 표시한다
- [ ] 트랜잭션 실패 시 에러 메시지를 표시한다

### F-21: 결과 표시
- [ ] Resolved 마켓에서 승리 방향(Yes/No)을 명확히 시각적으로 표시한다
- [ ] 최종 Yes/No 풀 비율을 표시한다
- [ ] Voided 마켓은 "무효 처리됨" 상태를 표시한다
- [ ] MarketResolved 이벤트를 리스닝하여 실시간 갱신한다

### F-22: 베팅 내역 조회
- [ ] "내 베팅" 탭에서 사용자가 참여한 마켓 목록을 표시한다
- [ ] 각 항목에 마켓 질문, 베팅 방향, 금액, 현재 배당률, 마켓 상태를 표시한다
- [ ] Active/Closed/Resolved/Voided 상태별로 구분한다
- [ ] 클레임 가능한 항목에 "클레임" 버튼을 표시한다

### F-23: 정산 내역
- [ ] 클레임 완료된 항목에 보상 금액, 순수익(보상-원금)을 표시한다
- [ ] 패배한 항목에 손실 금액을 표시한다
- [ ] 환불된 항목에 환불 금액을 표시한다

### F-24: 참여 이력
- [ ] 프로필 페이지에서 전체 마켓 참여 기록을 표시한다
- [ ] 승리/패배/무효 건수를 요약 표시한다
- [ ] 시간순으로 정렬된 참여 이력 목록을 표시한다

### F-25: 수익률 대시보드
- [ ] 총 베팅 금액, 총 수익 금액, 순이익을 표시한다
- [ ] 승률(승리 건수 / 전체 확정 건수)을 백분율로 표시한다
- [ ] 평균 베팅 금액, 최대 단일 수익을 표시한다

### F-26: 리더보드
- [ ] 수익률/정확도 기준으로 상위 예측자를 랭킹 표시한다
- [ ] 지갑 주소(축약), 승률, 총 수익, 참여 마켓 수를 표시한다
- [ ] 이벤트 로그 기반으로 온체인 데이터를 집계한다

### F-27: 4개 언어 지원
- [ ] 한국어/영어/중국어/일본어 UI를 지원한다
- [ ] 헤더에 언어 선택 드롭다운을 제공한다
- [ ] 브라우저 언어 설정을 기본값으로 감지한다
- [ ] 선택한 언어를 localStorage에 저장한다

### F-28: 통화 로컬라이제이션
- [ ] META 금액 옆에 선택된 법정화폐(KRW/USD/CNY/JPY) 환산값을 참고 표시한다
- [ ] 환율은 외부 API 또는 하드코딩으로 제공한다
- [ ] 언어 설정에 따라 기본 통화가 자동 선택된다

### F-29: 마켓 질문 다국어
- [ ] 마켓 상세에서 사용자의 언어 설정에 맞는 질문을 표시한다
- [ ] 해당 언어 질문이 없으면 영어(기본)로 폴백한다
- [ ] 컨트랙트의 question/questionKo/questionZh/questionJa 필드를 활용한다

### F-30: 정렬 옵션
- [ ] 마감 임박순/풀 규모순/최신순/인기순(참여자 수) 정렬을 제공한다
- [ ] 기본 정렬은 마감 임박순이다
- [ ] 정렬 선택 UI를 드롭다운으로 제공한다

### F-31: 풀 비율 차트
- [ ] 마켓 상세에서 Yes/No 풀 비율 변화를 시계열 차트로 표시한다
- [ ] BetPlaced 이벤트 로그를 조회하여 시간대별 풀 변화를 계산한다
- [ ] 라인 차트 또는 영역 차트로 시각화한다

### F-32: 마켓 검색
- [ ] 키워드 입력으로 마켓 질문을 검색한다
- [ ] 프론트엔드에서 로드된 마켓 목록을 필터링한다 (온체인 검색 없음)
- [ ] 검색 결과가 없을 때 안내 메시지를 표시한다
