# 프로젝트 로드맵

## Milestone 0: 프로젝트 기반 구축
- **목표**: 개발 환경 설정, 시스템 설계 문서 작성
- **작업**: Hardhat 프로젝트 초기화, React 프로젝트 스캐폴딩, 시스템 설계(/init-dev + /design)
- **완료 기준**: Hardhat 컴파일/테스트 환경 동작, React 개발 서버 기동, 설계 문서 완료

## Milestone 1: 스마트 컨트랙트 핵심
- **목표**: 예측 마켓의 핵심 라이프사이클(생성 -> 베팅 -> 결과 확정 -> 클레임/환불)을 스마트 컨트랙트로 구현
- **기능**: F-01, F-02, F-03, F-04, F-05, F-06, F-07, F-08
- **병렬 그룹**:
  - M1-A: F-01(마켓 생성), F-05(Yes/No 베팅) — 독립 개발 후 통합
  - M1-B: F-07(보상 클레임), F-08(Void 환불) — F-03/F-04 완료 후 병렬 진행
- **완료 기준**:
  - 마켓 생성/카테고리/결과 확정/무효화 전체 플로우 테스트 통과
  - Yes/No 베팅 + 추가 베팅 테스트 통과
  - 보상 계산 정확성 검증 (PRD 예시 시나리오 일치)
  - Void 환불 전액 반환 검증
  - Testnet 배포 및 기본 동작 확인

## Milestone 2: 스마트 컨트랙트 확장
- **목표**: 이의제기 시스템, 긴급 중단, 설정 관리 등 운영/거버넌스 기능 추가
- **기능**: F-09, F-10, F-11, F-12
- **병렬 그룹**:
  - M2-A: F-09(이의제기 기간), F-10(이의제기) — 상호 의존이나 순차 개발
  - M2-B: F-11(긴급 중단), F-12(설정 관리) — M2-A와 병렬 진행
- **완료 기준**:
  - 이의제기 기간 24h 적용 및 임계값 재심 전환 테스트
  - 이의제기 스테이킹/반환/몰수 테스트
  - 마켓 Pause/Resume 플로우 테스트
  - 설정 변경 및 수수료 인출 테스트
  - Testnet 재배포 및 통합 테스트

## Milestone 3: 프론트엔드 핵심
- **목표**: 지갑 연결부터 마켓 탐색, 베팅, 결과 확인, 클레임까지 핵심 사용자 플로우 구현
- **기능**: F-13, F-14, F-15, F-16, F-17, F-18, F-19, F-20, F-21, F-22, F-23
- **병렬 그룹**:
  - M3-A: F-13(지갑 연결), F-14(마켓 목록) — 기반 UI 병렬 구현
  - M3-B: F-18(베팅 수량 입력), F-19(예상 수익 표시) — 베팅 UI 병렬 구현
- **완료 기준**:
  - MetaMask 연결 및 Metadium 네트워크 전환 동작
  - 마켓 목록/상세/카테고리 필터 정상 표시
  - 베팅 플로우(금액 입력 -> 확인 모달 -> MetaMask 서명 -> 성공) 동작
  - 실시간 배당률 갱신 동작
  - 결과 표시 및 클레임/환불 UI 동작
  - 베팅/정산 내역 조회 동작

## Milestone 4: 프론트엔드 확장
- **목표**: 프로필/통계, 다국어, 검색/정렬 등 사용자 경험 확장 기능 구현
- **기능**: F-24, F-25, F-26, F-27, F-28, F-29, F-30, F-31, F-32
- **병렬 그룹**:
  - M4-A: F-24(참여 이력), F-25(수익률 대시보드) — 프로필 UI 병렬
  - M4-B: F-27(4개 언어 지원), F-28(통화 로컬라이제이션) — i18n 병렬
  - M4-C: F-30(정렬 옵션), F-31(풀 비율 차트), F-32(마켓 검색) — 탐색 확장 병렬
- **완료 기준**:
  - 프로필 페이지에서 참여 이력 및 수익률 통계 정상 표시
  - 리더보드 랭킹 표시
  - 4개 언어 전환 동작 및 언어별 마켓 질문 표시
  - 통화 환산 표시
  - 정렬/검색/풀 차트 동작

## Milestone 5: 통합 테스트 및 Testnet 배포
- **목표**: 전체 기능 통합 테스트, Testnet 최종 검증
- **작업**: E2E 테스트, 보안 검증(ReentrancyGuard, 오버플로우 등), Testnet 배포 및 검증
- **완료 기준**:
  - 전체 사용자 플로우 E2E 테스트 통과
  - 스마트 컨트랙트 보안 체크리스트 완료
  - Testnet 최종 배포 및 초기 마켓 생성/베팅/클레임 플로우 검증
  - 프론트엔드 스테이징 배포 (Testnet 연동)
