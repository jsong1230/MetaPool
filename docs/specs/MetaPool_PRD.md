## 1. 제품 개요

### 1.1 한 줄 요약
MetaPool은 Metadium 블록체인 위에서 META 토큰을 사용하여 다양한 주제(암호화폐, 스포츠, 날씨, 선거 등)의 Binary(Yes/No) 예측에 참여하고, 정확한 예측자가 보상을 받는 탈중앙화 예측 마켓 플랫폼이다.

### 1.2 비전
"누구나 의견을 걸 수 있는 글로벌 예측 시장" — 복잡한 금융 상품이 아닌, 간단한 Yes/No 베팅으로 진입 장벽을 낮추고, 블록체인의 투명성으로 신뢰를 확보한다.

### 1.3 배경 및 기회
Polymarket이 2024년 미국 대선 기간 월간 거래량 $1B+를 기록하며 예측 마켓의 시장성을 증명했다. 그러나 대부분의 예측 마켓은 Ethereum/Polygon 기반으로 높은 가스비 또는 복잡한 브릿지 과정을 요구한다. MetaPool은 Metadium의 저렴한 가스비와 META 토큰의 낮은 단가(~14 KRW)를 활용하여, 소액으로도 부담 없이 참여 가능한 예측 마켓을 구현한다.

### 1.4 경쟁 분석

| 항목 | Polymarket | Augur v2 | Azuro | **MetaPool** |
|------|-----------|----------|-------|-------------|
| 체인 | Polygon | Ethereum | Gnosis/Polygon | **Metadium** |
| 토큰 | USDC | ETH/DAI | USDT | **META** |
| 마켓 타입 | Binary + Multi | Binary + Scalar | 스포츠 특화 | **Binary** |
| 가스비 | 중 | 고 | 저 | **극저** |
| 오라클 | UMA | REP 투표 | 자체 | **관리자+커뮤니티** |
| 최소 참여금액 | ~$1 | ~$5 | ~$1 | **100 META (~$0.11)** |
| 결과 확정 | 분쟁 시스템 | 투표 시스템 | 자체 피드 | **관리자 확정 + 이의제기** |

### 1.5 MetaLotto와의 시너지
MetaLotto 사용자가 이미 MetaMask + Metadium 네트워크 + META 토큰을 보유하고 있으므로, MetaPool 온보딩 마찰이 0에 가깝다. 두 제품 간 크로스 프로모션(로또 당첨금으로 예측 마켓 참여 등)이 가능하다.

## 4. 핵심 컨셉

### 4.1 마켓 라이프사이클

```
[생성] → [베팅 진행] → [베팅 마감] → [이벤트 발생] → [결과 확정] → [정산]
 관리자    사용자      자동/수동     현실 세계     관리자/오라클   스마트 컨트랙트
```

### 4.2 Binary 마켓 구조

하나의 마켓은 하나의 질문과 두 가지 선택지(Yes/No)로 구성된다.

**예시 마켓:**
- "2026년 3월 31일까지 BTC가 $100,000을 돌파할까?" → Yes / No
- "다음 토트넘 vs 아스날 경기에서 토트넘이 이길까?" → Yes / No
- "서울 내일 최고 기온이 20°C를 넘을까?" → Yes / No

### 4.3 보상 분배 구조

```
총 베팅 풀 = Yes 풀 + No 풀

결과가 Yes인 경우:
  승리 풀 = Yes 풀
  패배 풀 = No 풀

플랫폼 수수료 = 패배 풀 × 2%
분배 가능 금액 = 패배 풀 - 플랫폼 수수료

각 승리자 보상 = 원금 + (분배 가능 금액 × 내 베팅 / 승리 풀)
```

**예시:**
- Yes 풀: 10,000 META (A가 6,000 / B가 4,000)
- No 풀: 15,000 META
- 결과: Yes
- 플랫폼 수수료: 15,000 × 2% = 300 META
- 분배 가능: 14,700 META
- A 보상: 6,000 + (14,700 × 6,000/10,000) = 6,000 + 8,820 = **14,820 META**
- B 보상: 4,000 + (14,700 × 4,000/10,000) = 4,000 + 5,880 = **9,880 META**

### 4.4 오즈(배당률) 표시

실시간 배당률을 표시하여 사용자가 기대 수익을 쉽게 파악할 수 있다.

```
Yes 확률 = Yes 풀 / 총 풀 × 100
No 확률 = No 풀 / 총 풀 × 100

Yes 배당률 = 총 풀 / Yes 풀
No 배당률 = 총 풀 / No 풀
```

**예시:** Yes 풀 40%, No 풀 60% → Yes 배당 2.5x, No 배당 1.67x

## 5. 기능 요구사항

### 5.1 우선순위 정의
- **P0**: v1 런칭 필수 (Must Have)
- **P1**: v1 포함 권장 (Should Have)
- **P2**: v1.1 이후 (Nice to Have)

### 5.2 마켓 관리 (관리자)

| ID | 기능 | 우선순위 | 설명 |
|----|------|---------|------|
| M-1 | 마켓 생성 | P0 | 관리자가 질문, 선택지(Yes/No), 베팅 마감 시간, 결과 확정 예정 시간을 설정하여 마켓 생성 |
| M-2 | 마켓 카테고리 | P0 | Crypto, Sports, Weather, Politics, Entertainment, Other 6개 카테고리 |
| M-3 | 결과 확정 | P0 | 관리자가 이벤트 결과를 Yes/No/무효(Void)로 확정 |
| M-4 | 이의제기 기간 | P1 | 결과 확정 후 24시간 이의제기 기간, 이의 수가 임계값 초과 시 재심 |
| M-5 | 마켓 무효화 | P0 | 이벤트 취소/불명확 시 마켓을 Void 처리하고 전액 환불 |
| M-6 | 긴급 마켓 중단 | P1 | 비정상 상황 발생 시 관리자가 마켓 베팅을 즉시 중단 |

### 7.3 마켓 카드 구성

각 마켓 카드는 다음 정보를 표시한다:
- 카테고리 태그 (Crypto/Sports/Weather 등)
- 질문 텍스트
- Yes/No 비율 프로그레스 바 (그린/레드)
- 총 풀 금액 (META)
- 참여자 수
- 마감 카운트다운
- Yes/No 배당률
- [Yes 베팅] [No 베팅] 버튼

### 7.4 베팅 플로우

```
마켓 카드에서 [Yes] 또는 [No] 클릭
→ 베팅 패널 슬라이드 오픈
  - 선택 방향 표시 (Yes/No)
  - 베팅 금액 입력 (슬라이더 + 직접입력)
  - 퀵 버튼: 100 / 500 / 1K / 5K / 10K META
  - 현재 배당률 표시
  - 예상 수익 실시간 계산
  - [베팅 확인] 버튼
→ 확인 모달
  - 마켓 질문
  - 내 선택: Yes/No
  - 베팅 금액: X META
  - 현재 배당률: X.XXx
  - 예상 수익: X META
  - [서명 & 베팅] [취소]
→ MetaMask 트랜잭션 서명
→ 성공 토스트
```

## 8. 마켓 카테고리 상세

### 8.1 카테고리 정의

| 카테고리 | 아이콘 | 설명 | 예시 질문 |
|---------|-------|------|----------|
| Crypto | 📈 | 암호화폐 가격, 시총, 이벤트 | "BTC $100K 돌파?" |
| Sports | ⚽ | 스포츠 경기 결과, 기록 | "토트넘 승리?" |
| Weather | 🌤️ | 날씨, 기온, 강수량 | "서울 내일 비 올까?" |
| Politics | 🏛️ | 선거, 정책, 국제 관계 | "조기총선 실시?" |
| Entertainment | 🎬 | 영화, 음악, 방송 | "넷플릭스 오징어게임 시즌3 1주차 1위?" |
| Other | 🔮 | 기타 모든 주제 | "일론 머스크 트윗할까?" |

### 8.2 결과 확정 기준

각 카테고리별 결과 확정 소스:
- **Crypto**: CoinGecko/CoinMarketCap API 가격 기준
- **Sports**: 공식 리그 기록
- **Weather**: 기상청/OpenWeather 공식 데이터
- **Politics**: 공식 발표/관보
- **Entertainment**: 공식 차트/플랫폼 데이터
- **Other**: 관리자 판단 + 이의제기 시스템

## 12. 부록

### 12.1 용어 정의

| 용어 | 설명 |
|------|------|
| 마켓(Market) | 하나의 예측 질문과 그에 대한 베팅 풀 |
| 베팅(Bet) | 특정 방향(Yes/No)에 META를 거는 행위 |
| 풀(Pool) | 한 방향에 모인 총 META 금액 |
| 배당률(Odds) | 현재 풀 비율에 기반한 기대 수익 배수 |
| 클레임(Claim) | 승리 후 보상을 인출하는 행위 |
| Void | 마켓 무효화, 전액 환불 |
| 이의제기(Dispute) | 결과에 동의하지 않을 때 제출하는 이의 |
